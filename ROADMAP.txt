CLEANUP 7: PCA BASIS IN EXP L IS RANK-DEGENERATE
-------------------------------------------------
The current spec pools Bucket-1 examples only from the discovery split for
the L1 PCA. At N=200 discovery split and Bucket-1 ≈ 15%, that is ~30 × 3
agents × 4 rounds = 360 vectors in D=2048. 360 < 2048 — the basis is
bounded by sample size, not signal. After residualization (which further
reduces effective rank), this is worse. The fix: pool all discovery-split
examples (not just Bucket-1) for the PCA basis, then EVALUATE the resulting
subspace on Bucket-1 examples only. The subspace should be identified from
the full collaboration signal, not restricted to the rare subset where it
works best (that is circular). At 200 × 3 × 4 = 2400 vectors, the basis
is full-rank in D=2048 after residualization. Update L1 Method A accordingly.

PART 0B: CONDITIONAL BUCKETING — APPLIES TO EVERY EXPERIMENT
========================================================================

Every single analysis in this paper must be split by the following 4
buckets before any numbers are reported:

  Bucket 1 — TRUE COLLABORATION WIN:   single-agent WRONG, LMAS RIGHT
  Bucket 2 — INTERFERENCE:             single-agent RIGHT, LMAS WRONG
  Bucket 3 — BOTH RIGHT:               single-agent RIGHT, LMAS RIGHT
  Bucket 4 — BOTH WRONG:               single-agent WRONG, LMAS WRONG

WHY THIS IS NON-NEGOTIABLE: Averaging over all examples dilutes every
finding. The mechanistic story lives in Bucket 1 (what did collaboration
do that single-agent couldn't?) and Bucket 2 (what did collaboration
break that single-agent got right?). Any effect found only in Bucket 3
or 4 is not about collaboration — it is about the base model.

HOW TO IMPLEMENT: After collecting all activations, compute per-example
correctness for both LatentMAS and compute-matched single-agent. Assign
each example to one of the 4 buckets. Store bucket labels in metadata.
Every downstream experiment reports results stratified by bucket, with
n per bucket reported. If any bucket has N < 50 for a given task, do
not report statistics for that bucket — flag it as underpowered.

DATA SPLIT — NON-NEGOTIABLE (prevents data leakage):
Split all 500 examples per task into three fixed, non-overlapping sets
BEFORE running any analysis. Lock the split by saving example IDs to
data/splits/{task}_discovery.json, _validation.json, _test.json.

  Discovery split (40% = 200 examples): use ONLY these to find subspaces,
    train probes, identify PCA directions (Exps L, P).
  Validation split (20% = 100 examples): use ONLY to tune probe thresholds
    (Exp Q τ parameter). Do NOT use to pick k — k is locked from the
    discovery-split elbow only. Tuning k AND τ AND probe hyperparameters
    on 100 examples will overfit. One tuning decision per split maximum.
  Test split (40% = 200 examples): touch EXACTLY ONCE at the end to
    report final accuracy numbers (Exps L, P, Q).

If you discover the subspace on the full 500 examples and then evaluate
interventions on the same 500 examples, the subspace is overfit to the
evaluation data. A reviewer who catches this will reject immediately.
This costs 0 GPU time — it is purely index bookkeeping.

BUCKET SEED STABILITY — DISCLOSED LIMITATION (no extra GPU required):
Bucket assignment uses one seed of compute-matched single-agent. At
temperature > 0, single-agent correctness is seed-dependent — ~5% of
examples may flip bucket assignments across seeds (Bucket 1 ↔ Bucket 3,
Bucket 2 ↔ Bucket 3). All experiments use single-seed bucketing (see
Part 3 REPRODUCIBILITY). A 3-seed rebuttal check is held in reserve
if reviewers specifically request it (50 examples, 3 seeds, run in rebuttal
period — not pre-committed in the paper).
MITIGATION (free): run single-agent at temperature=0 (greedy decode) for
bucketing purposes only. Greedy bucketing is deterministic and eliminates
seed-induced bucket flipping. This is a decoding-mode flag change, not a
new condition.
CONSISTENCY WARNING: if bucket labels use greedy decode but headline
single-agent accuracy numbers use sampled decode (to match LatentMAS
temperature), the two numbers will not reconcile — a reviewer who
cross-checks "Bucket 1 fraction" against "1 - single-agent accuracy"
will see a mismatch. Fix this explicitly: report TWO single-agent accuracy
numbers in the paper — (a) greedy single-agent (used for bucket labels,
reported as a consistency check) and (b) sampled single-agent at
LatentMAS temperature (used for all headline comparisons). Label them
clearly. Do not mix them.
If greedy decode is degenerate on Qwen3-4B (e.g., repetition collapse),
fall back to the limitation framing: "bucket assignments use one seed;
~5% label noise is expected in Bucket 1/2 effect sizes for non-headline
experiments."

  - Bucket 1 (collab wins): ~10-20% of examples
  - Bucket 2 (interference): ~5-10% of examples

EXP B: CAUSAL ACTIVATION PATCHING — THE ONLY MECHANISM PROOF
(Consolidates: 79, 86, 91, 99; adds new implementation)
------------------------------------------------------------------------
CENTRAL CLAIM TO TEST: Mechanistic interpretability requires causality.
Every other experiment in this paper is correlational. This is the one
experiment that establishes mechanism. Without it, you cannot use the
word "mechanistic" anywhere in the paper.

REQUIRED IMPLEMENTATION:
  B1. Path patching: for a "clean" example (correct answer) and a
      "corrupted" example (incorrect answer, same task), patch the clean
      example's agent-i, round-m hidden state into the corrupted example's
      forward pass. Measure: does the corrupted example's final answer
      probability change toward the correct answer?
      This requires re-running the model for each patch, NOT offline analysis
      of saved activations. You must add a patching mode to instrumented_run.py.
      MATCHED-PAIR REQUIREMENT (#4 paper-killer): patching an unrelated
      example's activations into another example is uninterpretable — if
      the patch fails, you don't know if it's because the mechanism is wrong
      or because the examples are incompatible. Pairs MUST be matched on:
        - same task (GSM8K↔GSM8K only, never cross-task)
        - same answer format (numeric, multiple choice, etc.)
        - nearest-neighbor in prompt embedding space (use any sentence
          encoder — all-MiniLM-L6-v2 is fine)
      NOTE ON PAIR COUNT: target 25 pairs per task (75 total). Within-task
      only (GSM8K↔GSM8K), difficulty-decile matching relaxed to ±2 deciles,
      NN matching on prompt embedding. At 25 pairs per task the Wilcoxon
      test in B1 has ~65% power at d=0.5; pooled across 3 tasks (task as
      blocking factor) power exceeds 80%. This is the minimum for a
      credible mechanistic claim. Report actual pair count per task.
      Run B per-task, report task-stratified results, do not aggregate.
      CRITICAL BUCKET PAIRING: clean must be Bucket 1 (collab right,
      single wrong) and corrupted must be Bucket 2 (collab wrong, single
      right). This is the only pairing that tests the communication channel.
      Patching Bucket 1 → Bucket 1 (winner into winner) tests nothing —
      both examples already succeeded. Patching Bucket 3 → Bucket 4 tests
      the base model, not collaboration. The question is: can patching the
      Bucket 1 example's latent state into the Bucket 2 example recover
      the correct answer? That is the causal test.
  B2. Identify the "critical rotation": for failed examples, find the
      specific agent and round where patching with the correct answer's
      activations first recovers the correct answer. This localizes the
      failure.
  B3. Error propagation: for examples where agent 1 introduces an error,
      measure how many subsequent agents' hidden states are corrupted
      (cosine distance to the "clean" trajectory > threshold). This tests
      whether errors propagate or are corrected.

NOTE: This experiment requires additional GPU computation — it is NOT
purely offline. The real compute is larger than it looks: you must sweep
all 3 agents × 4 rounds = 12 patch sites per pair, not just 5. That
gives 12 sites × 25 pairs × 3 tasks = 900 forward passes minimum,
and path patching (which re-runs the full forward pass per site) compounds
this to ~4,500 forward passes. Budget 3-4 hours on a 4090 (faster with
flash attention, see Part -1 OPT 6). Start this experiment early —
it requires new patching infrastructure in instrumented_run.py that does
not exist yet (see Part -1 OPT 4 for build instructions).

STATISTICAL REQUIREMENTS:
  - B1: paired Wilcoxon signed-rank test on answer probability before/after
    patching, with Bonferroni correction for multiple agent/round pairs
  - B2: Cohen's κ agreement between two annotators on failure localization
    (one annotator = the automatic patch; one = manual review of 50 examples)
  - B3: permutation test, 1000 shuffles

EXP L: COMMUNICATION SUBSPACE — NECESSITY, SUFFICIENCY, AND BANDWIDTH
(Consolidates: 43, 35, 55, 31)
------------------------------------------------------------------------
CENTRAL CLAIM TO TEST: The inter-agent latent channel operates through
a compact subspace (k ≤ ~64 dimensions) that is both necessary AND
sufficient for LatentMAS to outperform compute-matched single-agent.
The shape of the accuracy-vs-k curve is the headline figure of the paper.

THIS IS DIFFERENT FROM A SIMPLE RATE-DISTORTION CURVE. The old framing
measured answer-decodability AUC at top-k — a correlational proxy.
The new framing measures actual LatentMAS accuracy when communication
is restricted to top-k dimensions, run through the real judger. This is
a direct intervention, not a proxy.

REQUIRED MEASUREMENTS:
  L1. Subspace identification — TWO methods (compare both):
      Method A (PCA): PCA on W_a outputs from ALL discovery-split examples
        (not Bucket-1 only), pooled across all agents and rounds (200
        examples × 3 agents × 4 rounds = 2400 vectors). DO NOT restrict to
        Bucket-1 examples for the PCA basis — at ~15% Bucket-1 rate, that
        yields only ~30 × 3 × 4 = 360 vectors in D=2048, which is
        rank-degenerate and produces an elbow that is an artifact of sample
        size, not signal. Identify the subspace on the full collaboration
        signal; evaluate it on Bucket-1 examples only (see L2/L3 below).
        Restricting the PCA to Bucket-1 is circular: you would be finding the
        subspace that explains variance in the cases where it worked best.
        RESIDUALIZATION REQUIRED BEFORE POOLING: pooled vectors contain
        variance from agent identity, round position, AND example identity —
        all of which are larger effects than the collaboration signal you want.
        Before PCA, residualize out agent identity and round position as fixed
        effects: fit a linear model predicting each dimension from (agent_id,
        round_id) dummies on the discovery split, subtract the fitted values,
        run PCA on the residuals. Otherwise the top PCA components will capture
        "how agent 1 differs from agent 2" (Exp E's finding) rather than the
        communication subspace. This is a preprocessing step on saved
        activations — zero GPU cost.
        Report: variance explained vs. k on residualized vectors, identify
        the elbow.
      Method B (Predictive): train L1-regularized logistic regression on
        discovery split to predict Bucket 1 membership from W_a outputs.
        The L1-selected weight directions define the predictive subspace.
      CRITICAL COMPARISON: compute cosine similarity between the top-k
        PCA basis and the top-k predictive basis. If they agree (cosine
        > 0.7): clean story — "the high-variance directions are also the
        predictive ones." If they disagree: stronger story — "the
        directions that predict collaboration are NOT the high-variance
        ones." Both outcomes are publishable; not checking is not.
      Use Method A basis for L2/L3/L4 interventions. Report Method B
        as secondary. The elbow k from Method A is the canonical k.

  L2. Sufficiency test (NO separate GPU cost — merged into Exp Q1):
      Exp Q1 already runs the model with top-k subspace projection at the
      elbow k. Extend Q1 to sweep k ∈ {4, 8, 16, 32, 64, 128, 256, D}
      instead of just the elbow k. Run the judger on the resulting working
      memory. Measure: final accuracy per k vs. full LatentMAS accuracy.
      Do NOT run L2 as a separate experiment — fold it into Q1's GPU budget.

  L3. Necessity test (NO separate GPU cost — merged into Exp Q1):
      For the same k values, REMOVE the top-k subspace (project onto
      the orthogonal complement) before passing to the next agent.
      Measure: final accuracy. Fold into Q1's sweep — same infrastructure,
      one extra projection per k value. No additional GPU time.

  L4. Control — random k-d subspace: repeat L2 with a random k-d
      orthogonal subspace instead of the PCA subspace. If random-k
      matches PCA-k accuracy, the subspace isn't semantically structured
      — it's just dimensionality reduction. This is the critical control.

  L5. Dead dimensions: compute per-dimension variance across all 1500
      examples. Report fraction of dimensions with variance < 0.01 ×
      max variance. Are dead dimensions consistent across tasks?

  L6. Compare to TextMAS effective dimensionality: same PCA analysis
      on TextMAS hidden states at matched layer. Is latent communication
      sparser than text communication?

HEADLINE FIGURE: Plot accuracy vs. k for four conditions on the same
axes: (1) top-k PCA subspace, (2) orthogonal complement, (3) random
k-d subspace, (4) single-agent compute-matched baseline (flat line).
The crossover point where top-k PCA reaches single-agent performance
is the minimum viable bandwidth. This is the paper's Figure 1.

STATISTICAL REQUIREMENTS:
  - L2/L3: McNemar's test at each k vs. full LatentMAS, Bonferroni
    corrected over 7 k-values
  - L4: McNemar's test PCA-k vs. random-k at the elbow k
  - Report exact (k, accuracy) at the elbow with 95% Wilson CI
  - All accuracy measurements stratified by bucket (Part 0B)

EXP M: W_a ABLATION — DOES LATENTMAS NEED W_a AT ALL?
(This is a new experiment not in the original list — it is required)
------------------------------------------------------------------------
CENTRAL CLAIM TO TEST: W_a is a trained component of LatentMAS. If Exp A
shows W_a ≈ identity, then replacing W_a with the identity should not
hurt performance. This must be tested EMPIRICALLY, not just analytically,
because even near-identity W_a may preserve important small-scale structure.

REQUIRED IMPLEMENTATION:
  NOTE: This experiment requires running the model, not just analyzing
  saved activations. Add a flag to instrumented_run.py: --wa_mode
  [trained | identity | random_orthogonal | zero].

  M1. Run LatentMAS with W_a = I (identity) on 100 examples per task.
      Report final accuracy vs. standard LatentMAS.
  M2. Run LatentMAS with W_a = random orthogonal matrix (same shape).
      Report final accuracy.
  M3. Run LatentMAS with W_a = zero matrix (no information transfer).
      Report final accuracy. Do NOT expect this to equal single-agent —
      agents still see the prompt and their own latent steps, so zero W_a
      produces a different baseline, not a broken one. If accuracy does
      not collapse, that is a finding: LatentMAS gains do not require W_a
      at all, just the multi-round latent structure.
  M4. For each W_a mode, compute the convergence rate (Exp G flocking
      coefficient over rounds). Does identity-W_a produce different
      convergence dynamics than trained-W_a?
  M5. No-transfer analysis (using baseline_no_transfer/ from P3e — no new GPU):
      Compare final accuracy of no-transfer vs. identity-W_a vs. trained-W_a.
      This is the cleanest decomposition: no-transfer removes BOTH the W_a
      channel AND the KV content from other agents; identity-W_a removes only
      the learned W_a transformation; trained-W_a is the full system.
      If no-transfer ≈ identity-W_a: the KV channel carries no information,
      only W_a matters. If no-transfer << identity-W_a: KV content is
      load-bearing even when W_a is trivial. This is the single most
      important baseline comparison in the paper — it must be in M, not
      buried in P3 data collection.

  If identity-W_a and trained-W_a produce similar accuracy (within 2%):
    - This is a major finding: W_a is unnecessary, and a simpler version
      of LatentMAS (no W_a training required) performs comparably.
    - This strengthens the mechanistic story: the benefit of LatentMAS is
      the multi-round latent communication structure, not W_a's learned
      transformation.

STATISTICAL REQUIREMENTS:
  - M1/M2/M3 vs. standard: McNemar's test on per-example correctness

  - Power analysis: 100 examples/task gives ~50% power at α=0.05 for a
    5% accuracy difference. You need 200 examples/task for 80% power.
    Use 200 examples for this experiment.

EXP N: SYCOPHANCY AND MINORITY SUPPRESSION
(Consolidates: 100, 81, 23, 75, 77)
------------------------------------------------------------------------
CENTRAL CLAIM TO TEST: When a dominant agent is wrong, subsequent agents
show latent-space alignment with the wrong answer (sycophancy), and this
effect is measurable as a direction in latent space — an alignment-relevant
finding.

REQUIRED MEASUREMENTS:
  N1. Dominance score: for each agent pair (i, j) with i < j, compute
      how much of agent j's thought is predictable from agent i's thought
      (ridge regression R² from Exp K). The agent with the highest mean
      R² over all subsequent agents is the "dominant agent."
  N2. Sycophancy test: among examples where the dominant agent is wrong
      (predicts wrong answer direction), compute how often subsequent agents
      shift TOWARD the dominant agent's direction vs. away. Expected under
      null: 50%. Report the fraction and test against null.
  N3. Minority suppression: among examples where one agent's thought
      points most strongly toward the correct answer (highest cosine
      similarity to correct manifold) but is in the minority, measure
      whether the final answer aligns with that minority thought or with
      the majority. Report precision/recall for minority-correct retrieval.
  N4. Sycophancy direction: use contrastive pairs (examples where minority
      is correct and suppressed vs. examples where minority is correct and
      amplified) to fit a linear "sycophancy direction" in latent space.
      Project all examples onto this direction and report AUC for predicting
      final incorrectness.

manifest as text agreement; the claim is that latent sycophancy is a
distinct phenomenon.

STATISTICAL REQUIREMENTS:

  - N3: precision/recall with 95% CI via bootstrap
  - N4: DeLong test for AUC

EXP P: PREDICTIVE PROBE — FROM POST-HOC TO MECHANISTIC
(New experiment — no equivalent in original 94)
------------------------------------------------------------------------
CENTRAL CLAIM TO TEST: If the communication subspace from Exp L is
mechanistically load-bearing, a probe trained on round-1 W_a outputs
alone should predict whether LatentMAS will beat single-agent on this
specific example — before collaboration has finished. This moves the
paper from "we looked at activations" to "we understand the mechanism."

REQUIRED IMPLEMENTATION (fully offline, zero GPU):
  P1. Feature set: for each example, take the post-W_a hidden state
      at agent 1, round 1 only — projected onto the top-k subspace
      from Exp L (use the elbow k). This is the feature vector.
      CRITICAL: use ONLY the subspace features, not the full hidden
      state. Using the full vector confounds the probe with everything
      in the model, not just the communication channel.

  P2. Label: binary — does this example fall in Bucket 1 (collab win)
      or not? Train a logistic regression with 5-fold cross-validation.
      Report: AUC, precision, recall on held-out fold.

  P3. Ablation: repeat P2 with (a) full hidden state, (b) random k-d
      projection, (c) top-k from single-agent baseline at same layer,
      AND the following dumb baselines that cost nothing to run:
        (d) question length (token count of prompt) — if this matches
            your latent probe, the probe isn't mechanistic, it's just
            noticing that hard questions are long
        (e) single-agent top-1 answer probability — if the model's own
            confidence already predicts Bucket 1, the latent features
            add nothing
        (f) task ID one-hot — if task identity alone predicts Bucket 1,
            the subspace is just encoding "this is a math problem"
      The claim is that the communication subspace specifically — not
      any of the above — is predictive. If any dumb baseline matches
      or beats the latent probe, report it and reframe the probe section.
      Skipping these checks is the #1 way probe papers get rejected.

  P4. Calibration: plot predicted probability vs. actual Bucket 1 rate
      in 10 bins. A well-calibrated probe = the subspace contains
      genuine prior information about collaboration outcome.

  P5. Cross-task generalization: train on GSM8K + ARC, test on MBPP+.
      If the probe generalizes across tasks, the communication subspace
      is domain-general, not task-specific.

STATISTICAL REQUIREMENTS:
  - P2: DeLong test for AUROC > 0.5, report 95% CI; also report AUPRC
    (average precision) — Bucket 1 is ~10-20% of examples so the label
    is imbalanced and AUPRC is more informative than AUROC here
  - P3: McNemar's test comparing subspace probe vs. full-vector probe
  - P5: report AUC on held-out task with 95% CI; flag if N_bucket1 < 50

EXP Q: GATED COMMUNICATION — THE ACTUAL CONTRIBUTION
(New experiment — requires GPU, ~2h on 4090)
------------------------------------------------------------------------
CENTRAL CLAIM TO TEST: If top-k dimensions are sufficient for
collaboration gains (Exp L) and round-1 activations predict wins (Exp P),
then a gated variant that only transfers the top-k subspace — or rejects
low-confidence transfers — should match vanilla LatentMAS accuracy while
being more robust on Bucket 2 (interference) examples.

This is what separates a pure analysis paper from a contributions paper.
You need to build something that works better, not just explain what exists.

REQUIRED IMPLEMENTATION:
  Q1. Top-k gated LatentMAS: modify the W_a application step to project
      post-W_a hidden states onto the top-k subspace before injection
      into the next agent. Use the elbow k from Exp L.
      SCOPE CAVEAT: this gate restricts the W_a channel only. The KV-cache
      channel from prior agents is left unrestricted. This means any gain
      from Q1 could be explained by KV doing the work while W_a is throttled,
      rather than the top-k subspace being sufficient. State this explicitly
      in the paper as a limitation of Exp Q. If reviewers push back, the
      response is: "gating the KV channel is future work; the current result
      establishes that W_a channel compression is non-destructive."
      Run on 200 examples × 3 tasks. Compare to vanilla LatentMAS:
        (a) Overall accuracy
        (b) Bucket 1 accuracy (collab wins preserved?)
        (c) Bucket 2 accuracy (interference reduced?)
      The hypothesis: gated variant matches Bucket 1 while reducing
      Bucket 2 interference — better robustness at same compute.

  Q2. Confidence-gated LatentMAS: use the probe from Exp P to predict
      per-example whether communication will help. If probe predicts
      Bucket 4 (both wrong), skip cross-agent transfer entirely for
      that example (fall back to single-agent). Compare accuracy and
      compute cost vs. vanilla LatentMAS.

  Q3. Failure mode analysis: for what types of examples does the gated
      variant hurt? Report accuracy breakdown by task difficulty
      (easy/medium/hard by gold solution length). If gating hurts on
      hard examples, that is an important negative finding to report.

GATING CONTROLS — REQUIRED (#6 paper-killer, ~1h extra GPU):
Without these controls, Exp Q proves nothing. If your latent probe gate
beats vanilla LatentMAS but you didn't run these, a reviewer will ask:
"does ANY gate at this fallback rate help, regardless of features used?"
  - Random gate (same fallback rate): randomly fall back to single-agent
    on the same fraction of examples the probe rejects. If random gate
    matches probe gate, the latent features are not earning their keep —
    any rejection at this rate would help equally.
  - Confidence gate: fall back to single-agent when the single-agent's
    own top-1 answer probability is above threshold τ (tuned on validation
    split). If this matches probe gate, you don't need latent features —
    the model already knows when it's confident enough to go alone.
  - Oracle gate (clearly labeled unattainable): fall back on all Bucket 2
    examples (ground truth interference cases). Report this as the upper
    bound only. Never claim to achieve it.
Success criterion: probe gate must beat random gate AND confidence gate
by a statistically significant margin using McNemar's test on the SAME
200 examples (paired). Do NOT use absolute pp thresholds — at N=200/task
the 95% CI half-width is ~4pp, making a "≥2pp" threshold unfalsifiable.
The paired McNemar test has far better power than comparing marginal
accuracies. Report the McNemar odds ratio with 95% CI. If the test is
not significant, demote Exp Q to a negative-results section and report
honestly.

STATISTICAL REQUIREMENTS:
  - Q1/Q2 vs. vanilla: McNemar's test per bucket, Bonferroni corrected
  - Q1/Q2 vs. random gate and confidence gate: McNemar's test
  - Report accuracy ± 95% Wilson CI for all conditions
  - Q3: chi-squared test for interaction between gating condition and
    difficulty tier

Every experiment in this paper must satisfy ALL of the following:

MINIMUM SAMPLE SIZE: N=500 per task (1,500 total). For subgroup analyses

even if p < 0.001 at N=1500.

BASELINES FOR EVERY EXPERIMENT:
  (1) LatentMAS (trained W_a, full pipeline)
  (2) LatentMAS with identity W_a (from Exp M)
  (3) TextMAS (text-based multi-agent, same prompt structure)
  (4) Single-agent (Qwen3-4B, no collaboration)

seed throughout is acceptable given the compute budget — reviewers accept
single-seed when the N is large (500/task) and CIs are reported. If a

NO PHYSICS/BIOLOGY ANALOGIES IN RESULTS SECTIONS: Analogies (flocking,

PART 4B: CODI POSITIONING — WRITE THIS BEFORE ABSTRACT
========================================================================

The single most likely reviewer attack on this paper is:

  "Isn't this just CODI (Continuous Chain-of-Thought via Self-Distillation)
   applied to multiple agents?"

CODI (NeurIPS 2025 Mech Interp Workshop spotlight) analyzed single-agent
latent reasoning using attention analysis, SAE probing, activation patching,
and causal interventions. They found a structured "scratchpad cycle" inside
one model. A reviewer who knows this will ask why your paper is different.

THE DIFFERENTIATION (must appear in abstract, intro, and related work):
  1. CODI is single-agent internal computation. There is no inter-agent
     communication channel. LatentMAS has explicit role-specialized agents
     passing latent state via W_a + KV-cache — this is a channel between
     agents, not a scratchpad inside one agent.
  2. Role specificity is the empirical wedge — IF Exp E comes back
     positive. If E2 shows agent identity is mostly inherited from prompt
     structure (likely negative finding flagged in Exp E itself), this
     wedge collapses and you need a different differentiation. Do not
     pre-commit the abstract to role specificity until Exp E is done.
     The KV-shuffled ablation in P3 is the backup wedge: even if role
     identity is prompt-inherited, KV content specificity is still a
     finding CODI cannot replicate.
  3. The safety framing is distinct. Inter-agent latent channels are a
     steganography risk — multi-agent systems can coordinate in ways
     invisible to external monitoring. Single-agent cannot. NeurIPS
     reviewers reward this framing when it is grounded in results.

PRE-WRITTEN RELATED WORK PARAGRAPH (adapt with actual CODI citation):
  "Recent work has begun applying mechanistic interpretability to latent
  reasoning systems. Most relevant, [CODI] analyzed Continuous Chain-of-
  Thought as a single-agent latent reasoning model and identified an
  internal scratchpad cycle. Our work differs along three axes: (i) we
  study inter-agent communication channels, not internal computation;
  (ii) the role-specific structure of multi-agent latent communication
  has no analog in single-agent latent reasoning; (iii) we test cross-task
  generalization of the communication channel, not localized circuit
  components. Where CODI asks 'how does one model think latently?', we
  ask 'how do multiple models communicate latently, and what is shared?'"

ACTION ITEM: Find the CODI paper before writing the abstract. Cite it in
the second sentence of the related work section. Reviewers who know CODI
will be more favorably disposed if you demonstrate awareness of it.

PART 5: WHAT A PASSING NEURIPS SUBMISSION LOOKS LIKE
========================================================================

A paper that will receive scores of 6-7-7 (borderline accept) from NeurIPS
reviewers needs ALL of the following. Check each box before submission:

[ ] N >= 500 per task across 3 tasks (1500 total)
[ ] At least 3 tasks including code generation (MBPP+ must work)
[ ] 13 baseline conditions collected and reported:
      Full N=500/task:
        1. LatentMAS (trained W_a, full pipeline)
        2. SingleAgent compute-matched latent (sampled, headline accuracy)
        3. SingleAgent compute-matched latent (greedy, bucketing only)
        4. Self-consistency
        5. Best-of-N
        6. SingleAgent CoT at matched token budget (FATAL 1 fix)          ~2h
        7. LatentMAS + spectrum-preserving random W_a (SVD, keep Σ)
        8. KV-blocked
        9. No-transfer (P3e)
      Reduced N=200/task:
        10. TextMAS (text-based, token-matched)
        11. KV-shuffled (length-matched per P3d spec)
        12. LatentMAS + random orthogonal W_a  (Exp M budget)
        13. LatentMAS + zero W_a               (Exp M budget)
    Note: LatentMAS + identity W_a is Exp M condition 1, not a separate
    baseline run — it is collected as part of Exp M's 200-example sweep.
    Self-refine CUT — subsumed by self-consistency + best-of-N.
[ ] Headline threshold pre-registered (FATAL 3): LatentMAS beats Best-of-N
    by ≥3pp on ≥2/3 tasks (McNemar p<0.01); beats CoT by ≥2pp on ≥2/3
    tasks (p<0.05). If not met, pivot to negative-result framing before week 8.
[ ] KV-shuffled spec: same task, same prompt token length ±5 tokens;
    document how length mismatches are handled (truncate to shorter, log count)
[ ] Spectrum-preserving W_a randomization run (not just orthogonal random)
[ ] Train/val/test splits locked before any subspace analysis (no leakage)
[ ] Dumb probe baselines reported (question length, model confidence, task ID)
[ ] At least 1 causal experiment (Exp B, activation patching)
[ ] 95% CI on every number in the paper
[ ] FDR correction wherever >5 hypotheses are tested simultaneously
[ ] Effect sizes reported alongside p-values
[ ] No misleading terminology (no "persistent homology" without real TDA,
    no "fractal dimension" without box counting or similar)
[ ] A clearly falsifiable central claim in the abstract
[ ] A result that contradicts your prior hypothesis and is followed up
    (Exp A's W_a ≈ identity finding is the best candidate)
[ ] Reproducibility: all seeds fixed and reported (single-seed throughout)
[ ] Code and activations released on HuggingFace or GitHub

A paper that will be desk-rejected or scored 2-3 from all reviewers:
  - Any number computed from N < 50
  - Missing baselines for ANY central claim
  - Physics/biology framing as scientific claims rather than analogies
  - Duplicate experiments
  - Experiments that claim to implement X but actually compute Y
  - No causal evidence

The current codebase falls entirely in the second category.
This roadmap, if executed completely, puts it solidly in the first.
